# 청소년 창업 팀구성 캐릭터 진단 웹서비스 기획 요약

## 1. 프로젝트 개요

청소년 창업 교육 프로그램의 일환으로, 학생들이 온라인 설문을 통해 자신의 창업 성향을 진단하고, 그 결과에 따라 창업팀 내 적합한 역할과 캐릭터를 추천받는 웹서비스를 기획하고 있다.

서비스의 핵심은 단순한 성향검사가 아니라, 학생들이 재미있고 직관적으로 자신의 창업 역할을 이해할 수 있도록 “게임 캐릭터 소개 화면”처럼 결과를 보여주는 것이다.

## 2. 서비스 목적

이 웹서비스의 목적은 다음과 같다.

1. 청소년들이 창업팀 안에서 자신에게 맞는 역할을 쉽게 이해하게 한다.
2. 팀 구성 시 CEO, CTO, CMO 같은 어려운 직책을 쉽고 친근한 캐릭터로 설명한다.
3. 10개 설문 문항을 통해 학생의 창업 성향을 진단한다.
4. 결과 화면에서 캐릭터 이미지, 레이더 차트, 능력치, 성향 분석 리포트를 제공한다.
5. 향후 3~5명 팀 구성 시 학생들의 역할 분포를 참고할 수 있게 한다.

## 3. 대상

대상은 초등 고학년, 중학생, 고등학생 공통형으로 설정했다.

문체는 너무 유치하지 않되, 초등 고학년도 이해할 수 있도록 쉬운 표현을 사용한다.
창업 용어는 함께 제공하되, 반드시 쉬운 역할명과 연결해서 설명한다.

예시:

- CEO → 방향잡이 리더
- CTO → 기술 제작자
- CMO → 홍보 전략가

## 4. 팀 구성 기준

팀 구성은 3~5명을 기준으로 한다.

기본적으로 4명 1팀 구성을 가장 안정적인 형태로 본다.
다만 교육 현장 상황에 따라 3명, 5명 팀도 가능하게 설계한다.

### 팀 인원별 역할 배치

#### 3명 팀

| 인원 | 역할 |
|---|---|
| 1명 | CEO + CPO |
| 1명 | CTO |
| 1명 | CMO + CFO/COO |

#### 4명 팀

| 인원 | 역할 |
|---|---|
| 1명 | CEO |
| 1명 | CPO / PM |
| 1명 | CTO |
| 1명 | CMO + CFO/COO |

#### 5명 팀

| 인원 | 역할 |
|---|---|
| 1명 | CEO |
| 1명 | CPO / PM |
| 1명 | CTO |
| 1명 | CMO |
| 1명 | CFO / COO |

## 5. 확정된 기본 역할 5개

역할은 A안 5개 역할로 확정했다.

| 실제 직책 | 쉬운 역할명 | 핵심 역할 |
|---|---|---|
| CEO | 방향잡이 리더 | 팀의 목표를 정하고 전체 방향을 이끈다 |
| CPO / PM | 아이디어 설계자 | 문제를 찾고 아이디어와 서비스를 기획한다 |
| CTO | 기술 제작자 | 앱, 웹, 제품, 시제품을 만든다 |
| CMO | 홍보 전략가 | 고객에게 알리고 발표·홍보·마케팅을 담당한다 |
| CFO / COO | 숫자 운영가 | 예산, 일정, 자료, 준비물, 운영을 관리한다 |

CFO는 청소년 교육에서 단독 재무 역할로만 설명하기보다, COO와 결합해 “돈과 운영을 함께 관리하는 역할”로 설명하는 것이 적합하다고 판단했다.

## 6. 캐릭터 이름 방향

캐릭터 이름은 A안으로 확정했다.

방향은 다음과 같다.

- 귀엽고 쉬운 이름
- 아이들이 기억하기 쉬운 이름
- 직책의 어려움을 줄이는 이름
- 게임 캐릭터처럼 친근한 이름
- 각 역할의 성향이 바로 연상되는 이름

## 7. 확정된 캐릭터 5종

### 1) 캡틴 루미

| 항목 | 내용 |
|---|---|
| 실제 역할 | CEO |
| 쉬운 역할명 | 방향잡이 리더 |
| 캐릭터명 | 캡틴 루미 |
| 핵심 성향 | 리더십, 결단력, 팀조율 |
| 한 줄 설명 | 팀의 목표를 정하고 친구들을 이끄는 리더형 캐릭터 |

캡틴 루미는 팀의 방향을 잡고, 친구들의 의견을 모으며, 최종 결정을 도와주는 캐릭터이다.

### 2) 플래너 도도

| 항목 | 내용 |
|---|---|
| 실제 역할 | CPO / PM |
| 쉬운 역할명 | 아이디어 설계자 |
| 캐릭터명 | 플래너 도도 |
| 핵심 성향 | 기획력, 분석력, 구조화 |
| 한 줄 설명 | 문제를 찾고 아이디어를 구체적인 계획으로 바꾸는 기획형 캐릭터 |

플래너 도도는 고객의 불편함을 찾고, 아이디어를 사업계획이나 서비스 구조로 정리하는 캐릭터이다.

### 3) 메이커 테오

| 항목 | 내용 |
|---|---|
| 실제 역할 | CTO |
| 쉬운 역할명 | 기술 제작자 |
| 캐릭터명 | 메이커 테오 |
| 핵심 성향 | 제작력, 실행력, 실험정신 |
| 한 줄 설명 | 직접 만들고 실험하며 아이디어를 현실로 바꾸는 제작형 캐릭터 |

메이커 테오는 앱, 웹, 시제품, 모형 등을 직접 만들고 테스트하는 데 강한 캐릭터이다.

### 4) 스토리 모아

| 항목 | 내용 |
|---|---|
| 실제 역할 | CMO |
| 쉬운 역할명 | 홍보 전략가 |
| 캐릭터명 | 스토리 모아 |
| 핵심 성향 | 표현력, 홍보력, 설득력 |
| 한 줄 설명 | 아이디어를 매력적으로 알리고 사람들의 관심을 끄는 표현형 캐릭터 |

스토리 모아는 발표, 홍보, 디자인, 브랜드, 광고, 설명에 강한 캐릭터이다.

### 5) 체크 누리

| 항목 | 내용 |
|---|---|
| 실제 역할 | CFO·COO |
| 쉬운 역할명 | 숫자 운영가 |
| 캐릭터명 | 체크 누리 |
| 핵심 성향 | 운영력, 책임감, 꼼꼼함 |
| 한 줄 설명 | 일정, 예산, 자료, 준비를 관리하는 운영형 캐릭터 |

체크 누리는 팀 프로젝트의 일정, 준비물, 예산, 제출 자료, 체크리스트를 관리하는 캐릭터이다.

## 8. 설문 방식

설문 방식은 10문항 5지선다형으로 확정했다.

각 문항에는 5개의 선택지가 있고, 각 선택지는 5개 캐릭터 중 하나와 연결된다.

예를 들어 한 문항에서 리더형 선택지를 고르면 캡틴 루미 점수에 1점이 더해진다.

## 9. 점수 계산 방식

각 문항에서 선택한 답변에 따라 해당 캐릭터 점수를 1점씩 부여한다.

| 선택 캐릭터 | 점수 코드 | 증가 점수 |
|---|---|---|
| 캡틴 루미 | CEO | +1 |
| 플래너 도도 | CPO | +1 |
| 메이커 테오 | CTO | +1 |
| 스토리 모아 | CMO | +1 |
| 체크 누리 | COO | +1 |

총 10문항 응답 후 가장 높은 점수를 받은 캐릭터가 대표 캐릭터가 된다.
두 번째로 높은 점수의 캐릭터는 보조 성향으로 표시한다.

## 10. 동점 처리 방식

동점이 발생하면 억지로 하나만 선택하지 않는다.
공동 성향 또는 복합 성향으로 보여준다.

예시:

- 캡틴 루미 3점
- 스토리 모아 3점
- 플래너 도도 2점
- 메이커 테오 1점
- 체크 누리 1점

이 경우 결과는 다음처럼 표현한다.

“당신은 캡틴 루미형 리더 성향과 스토리 모아형 표현 성향이 함께 강합니다. 추천 역할은 CEO 또는 CMO입니다.”

## 11. 균형형 처리 방식

5개 캐릭터 점수가 모두 비슷하게 나오는 경우에는 균형형 창업가로 표시할 수 있다.

예시:

| 캐릭터 | 점수 |
|---|---:|
| 캡틴 루미 | 2 |
| 플래너 도도 | 2 |
| 메이커 테오 | 2 |
| 스토리 모아 | 2 |
| 체크 누리 | 2 |

결과 문구 예시:

“당신은 여러 역할에 고르게 관심이 있는 균형형 창업가입니다. 팀 상황에 따라 리더, 기획, 제작, 홍보, 운영 역할을 유연하게 맡을 수 있습니다.”

## 13. 설문 구현 시 주의사항

선택지 순서는 화면마다 무작위로 섞는 것이 좋다.
항상 같은 순서로 보여주면 학생들이 첫 번째 선택지만 반복 선택할 가능성이 있기 때문이다.

내부 데이터에는 선택지와 캐릭터 코드를 연결해두고, 화면 출력 시 순서만 랜덤 처리한다.

예시 데이터 구조:

```json
{
  "question": "새로운 팀 프로젝트를 시작할 때 나는?",
  "options": [
    {
      "text": "팀 목표를 정하고 친구들이 잘 움직이도록 돕고 싶다",
      "type": "CEO"
    },
    {
      "text": "어떤 문제를 해결할지 먼저 정리하고 싶다",
      "type": "CPO"
    }
  ]
}

## 14. 결과 리포트 기본 방향

결과 리포트는 “게임 캐릭터 프로필 카드”처럼 구성한다.

단순한 텍스트 결과지가 아니라, 학생이 자신의 캐릭터를 획득한 느낌을 받을 수 있게 만들어야 한다.

핵심 방향:

1. 캐릭터 이미지가 크게 보여야 한다.
2. 캐릭터 이름과 역할이 명확히 보여야 한다.
3. 레이더 차트로 5개 성향을 직관적으로 보여준다.
4. 강점, 성장 포인트, 추천 역할, 팀 궁합을 카드형으로 보여준다.
5. 게임 캐릭터 소개 화면처럼 화려하고 주목도 있게 구성한다.

## 15. 결과 리포트 필수 구성요소

결과 리포트에는 다음 요소를 포함한다.

| 순서 | 구성요소 | 내용 |
| --- | --- | --- |
| 1 | 결과 타이틀 | 나의 창업 캐릭터 결과 |
| 2 | 캐릭터 이미지 | 정해진 캐릭터 이미지 |
| 3 | 캐릭터 이름 | 캡틴 루미 등 |
| 4 | 창업 역할명 | CEO / 방향잡이 리더 |
| 5 | 한 줄 소개 | 캐릭터 성향 요약 |
| 6 | 키워드 배지 | 리더십, 결단력, 팀조율 등 |
| 7 | 레이더 차트 | 5개 성향 점수 시각화 |
| 8 | 능력치 점수 카드 | 리더십, 기획력, 제작력, 표현력, 운영력 |
| 9 | 대표 성향 | 1순위 캐릭터 |
| 10 | 보조 성향 | 2순위 캐릭터 |
| 11 | 강점 | 3개 내외 |
| 12 | 성장 포인트 | 2~3개 내외 |
| 13 | 잘 맞는 팀원 | 궁합 좋은 캐릭터 2명 |
| 14 | 추천 활동 | 팀장, 발표, 제작, 홍보 등 |
| 15 | 캐릭터 대사 | 말풍선형 한마디 |

## 16. 레이더 차트 설계

레이더 차트는 5개 축으로 구성한다.

| 축 이름 | 의미 | 연결 캐릭터 |
| --- | --- | --- |
| 리더십 | 팀을 이끌고 방향을 정하는 힘 | 캡틴 루미 |
| 기획력 | 문제를 찾고 아이디어를 구조화하는 힘 | 플래너 도도 |
| 제작력 | 직접 만들고 실험하는 힘 | 메이커 테오 |
| 표현력 | 발표, 홍보, 설득, 스토리 전달 능력 | 스토리 모아 |
| 운영력 | 일정, 예산, 자료, 준비를 관리하는 힘 | 체크 누리 |

점수는 내부적으로 각 캐릭터별 원점수를 계산한 후, 화면에서는 100점 만점 기준으로 환산해 보여주는 방식이 좋다.

예시:

| 항목 | 원점수 | 환산점수 |
| --- | --- | --- |
| 리더십 | 4 | 80 |
| 기획력 | 2 | 40 |
| 제작력 | 1 | 20 |
| 표현력 | 2 | 40 |
| 운영력 | 1 | 20 |

## 18. 캐릭터별 리포트 문안

### 1) 캡틴 루미

| 항목 | 내용 |
| --- | --- |
| 역할 | CEO |
| 쉬운 역할명 | 방향잡이 리더 |
| 한 줄 소개 | 목표를 세우고 팀을 한 방향으로 이끄는 리더형 캐릭터 |
| 핵심 키워드 | 리더십, 결단력, 팀조율 |

강점:

- 팀의 목표를 빠르게 정할 수 있다.
- 친구들의 의견을 모아 방향을 잡는 데 강하다.
- 발표나 최종 결정 상황에서 적극적이다.

성장 포인트:

- 혼자 너무 빨리 결정하지 않도록 주의한다.
- 조용한 팀원의 의견도 끝까지 듣는 연습이 필요하다.
- 세부 실행은 팀원과 함께 나누는 것이 중요하다.

잘 맞는 팀원:

- 플래너 도도
- 체크 누리

추천 활동:

- 팀장
- 발표 대표
- 최종 의사결정
- 역할 분배

캐릭터 대사:

“우리 팀의 방향은 내가 잡을게. 다 같이 해보자!”

### 2) 플래너 도도

| 항목 | 내용 |
| --- | --- |
| 역할 | CPO / PM |
| 쉬운 역할명 | 아이디어 설계자 |
| 한 줄 소개 | 문제를 찾고 아이디어를 구체적인 계획으로 바꾸는 기획형 캐릭터 |
| 핵심 키워드 | 분석력, 기획력, 구조화 |

강점:

- 아이디어를 정리하고 체계적으로 구성하는 능력이 좋다.
- 고객의 문제나 불편을 잘 발견한다.
- 발표 흐름이나 사업계획 구조를 잘 잡는다.

성장 포인트:

- 생각만 길어지고 실행이 늦어지지 않도록 주의한다.
- 실제 사용자 반응을 빠르게 확인해 보는 연습이 필요하다.
- 팀원들과 아이디어를 자주 공유하면 더 좋다.

잘 맞는 팀원:

- 캡틴 루미
- 메이커 테오

추천 활동:

- 사업 아이디어 설계
- 문제 정의
- 발표 내용 구성
- 기획서 작성

캐릭터 대사:

“좋은 아이디어는 문제를 제대로 이해하는 것에서 시작돼!”

### 3) 메이커 테오

| 항목 | 내용 |
| --- | --- |
| 역할 | CTO |
| 쉬운 역할명 | 기술 제작자 |
| 한 줄 소개 | 직접 만들고 실험하며 아이디어를 현실로 바꾸는 실행형 캐릭터 |
| 핵심 키워드 | 실행력, 제작력, 실험정신 |

강점:

- 말보다 직접 만들어 보며 해결하는 데 강하다.
- 시제품, 앱, 웹, 모형 제작에 적합하다.
- 문제를 테스트하며 빠르게 개선할 수 있다.

성장 포인트:

- 기술이나 제작에만 집중하지 않도록 주의한다.
- 사용자 입장에서 보기 쉽게 설명하는 연습이 필요하다.
- 일정과 협업 소통을 함께 챙기면 더 강해진다.

잘 맞는 팀원:

- 플래너 도도
- 스토리 모아

추천 활동:

- 프로토타입 제작
- 코딩
- 기능 테스트
- 시제품 개선

캐릭터 대사:

“직접 만들어 보면 더 좋은 답이 보여!”

### 4) 스토리 모아

| 항목 | 내용 |
| --- | --- |
| 역할 | CMO |
| 쉬운 역할명 | 홍보 전략가 |
| 한 줄 소개 | 아이디어를 매력적으로 보여 주고 사람들의 관심을 끄는 표현형 캐릭터 |
| 핵심 키워드 | 표현력, 홍보력, 설득력 |

강점:

- 발표, 홍보, 설명, 디자인에 강하다.
- 사람들의 관심을 끄는 표현 방식을 잘 찾는다.
- 브랜드 이름, 로고, 소개 문구 구성에 적합하다.

성장 포인트:

- 겉으로 보이는 부분만 강조하지 않도록 주의한다.
- 내용과 근거를 함께 준비하면 더 설득력 있어진다.
- 팀원들과 핵심 메시지를 맞추는 연습이 필요하다.

잘 맞는 팀원:

- 캡틴 루미
- 메이커 테오

추천 활동:

- 발표자
- 홍보물 제작
- 소개 문구 작성
- 브랜드 콘셉트 정리

캐릭터 대사:

“좋은 아이디어는 잘 보여줘야 더 빛나!”

### 5) 체크 누리

| 항목 | 내용 |
| --- | --- |
| 역할 | CFO·COO |
| 쉬운 역할명 | 숫자 운영가 |
| 한 줄 소개 | 일정, 자료, 예산, 준비를 꼼꼼하게 관리하는 운영형 캐릭터 |
| 핵심 키워드 | 꼼꼼함, 책임감, 운영력 |

강점:

- 준비물, 자료, 시간, 예산 관리를 잘한다.
- 빠진 부분을 점검하고 완성도를 높이는 데 강하다.
- 팀 프로젝트의 안정감을 높여 준다.

성장 포인트:

- 완벽하게 하려다 속도가 늦어지지 않도록 주의한다.
- 변화가 생길 때 유연하게 대응하는 연습이 필요하다.
- 숫자와 운영뿐 아니라 아이디어에도 관심을 가지면 더 좋다.

잘 맞는 팀원:

- 캡틴 루미
- 플래너 도도

추천 활동:

- 일정 관리
- 예산 정리
- 준비물 체크
- 제출 자료 관리

캐릭터 대사:

“작은 준비가 팀의 큰 성공을 만든다!”

## 3-3. 성향 분석 영역

첨부 기획에서는 레이더 차트를 5개 축으로 구성하고, 각 축은 리더십·기획력·제작력·표현력·운영력으로 설계되어 있습니다.

### 좌측: 레이더 차트

| 축 | 연결 캐릭터 | 의미 |
| --- | --- | --- |
| 리더십 | 캡틴 루미 | 팀을 이끌고 방향을 정하는 힘 |
| 기획력 | 플래너 도도 | 문제를 찾고 아이디어를 구조화하는 힘 |
| 제작력 | 메이커 테오 | 직접 만들고 실험하는 힘 |
| 표현력 | 스토리 모아 | 발표, 홍보, 설득, 스토리 전달 능력 |
| 운영력 | 체크 누리 | 일정, 예산, 자료, 준비를 관리하는 힘 |

### 우측: 능력치 점수 카드

```
나의 창업 능력치

리더십    80
기획력    40
제작력    20
표현력    60
운영력    40
```

### 점수 표시 방식

| 원점수 | 환산 점수 |
| --- | --- |
| 0점 | 0점 |
| 1점 | 20점 |
| 2점 | 40점 |
| 3점 | 60점 |
| 4점 | 80점 |
| 5점 이상 | 100점 |

단, 전체 문항이 10문항이므로 특정 캐릭터가 6점 이상을 받을 수 있습니다.

화면에서는 과도한 점수 쏠림을 방지하기 위해 **최대 100점으로 캡핑**하는 방식이 적절합니다.

```tsx
convertedScore = Math.min(rawScore * 20, 100)
```

# 10. 점수 계산 로직 설계

첨부 기획에서는 각 문항 선택지가 캐릭터 코드와 연결되고, 선택 시 해당 캐릭터에 +1점이 부여되는 구조로 정리되어 있습니다. 또한 최고 점수는 대표 캐릭터, 두 번째 점수는 보조 성향으로 표시하도록 되어 있습니다.

## 10-1. 기본 점수 계산

```tsx
const rawScores = {
  CEO: 0,
  CPO: 0,
  CTO: 0,
  CMO: 0,
  COO: 0
}

answers.forEach(answer => {
  rawScores[answer.type] += 1
})
```

---

## 10-2. 레이더 차트 점수 환산

```tsx
const convertedScores = {
  leadership: Math.min(rawScores.CEO * 20, 100),
  planning: Math.min(rawScores.CPO * 20, 100),
  making: Math.min(rawScores.CTO * 20, 100),
  expression: Math.min(rawScores.CMO * 20, 100),
  operation: Math.min(rawScores.COO * 20, 100)
}
```

---

## 10-3. 대표 캐릭터 산출

```tsx
const sortedScores = Object.entries(rawScores).sort(
  (a, b) => b[1] - a[1]
)

const primaryCode = sortedScores[0][0]
const secondaryCode = sortedScores[1][0]
```

---

## 10-4. 결과 유형 분기

| 유형 | 조건 | 표시 방식 |
| --- | --- | --- |
| 단일 대표형 | 1위 점수가 단독 최고 | 대표 캐릭터 1개 + 보조 성향 1개 |
| 복합 성향형 | 1위 점수가 2개 이상 동점 | 공동 대표 캐릭터 표시 |
| 균형형 | 5개 점수가 모두 유사 | 균형형 창업가 표시 |

### 균형형 판단 기준

```tsx
const scores = Object.values(rawScores)
const maxScore = Math.max(...scores)
const minScore = Math.min(...scores)

const isBalanced = maxScore - minScore <= 1
```
